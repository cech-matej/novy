Základní příkazy Gitu

Vytvoření a konfigurace lokálního repozitáře

git init

Vytvoření lokálního repozitáře

git --version

Vypíše verzi Gitu

git config --global http.proxy http://192.168.1.1:800

Nastaví přístup konzolové aplikaci přes proxy server (globálně, tj. pro všechny repozitáře vytvořené na daném počítači).

git config --global http.sslverify "false"

Tímto příkazem je možné pro daný počítač zrušit zabezpečenou komunikaci prostřednictvím https (SSL vrstvy), která vyžaduje přítomnost SSl certifikátu. Toto řešení lze použít např. pro případy, kdy na školních počítačích pracuje střídavě více žáků a nepoužívají vlastní uživatelský profil s uloženým certifikátem.

git config --local user.name "uživatelské jméno"
git config --local user.email "emailová adresa"

Nastaví uživatelský účet pro používání v lokálním repozitáři (vhodné pro školní počítače).

Práce s lokálním repozitářem

git status

Zjištění aktuálního stavu repozitáře - červeně zobrazeny změněné soubory, které nejsou součástí staging area, zeleně soubory ve staging area připravené na commit.

git add -all
git add .

Do staging area budou umístěny všechny soubory.

git add README.md

Do staging area bude umístěn soubor README.md.

git add *.jpg, *.png

Do staging area budou umístěny soubory (obrázky) s příponou jpg a png.

git reset

Ze staging area budou uvolněny všechny umístěné soubory.

git reset *.jpg

Ze stagging area budou uvolněny všechny soubory jpg.

Synchronizace lokálního a vzdáleného repozitáře

git remote add origin https://github.com/uživatelské jméno/název vzdáleného repozitáře.git

Připojí vzdálený repozitář pod názvem origin (obvykle zadané pojmenování), aby mohl být synchronizován s lokálním repozirářem.

git push -u origin master

Aktualizuje stav vzdáleného repozitáře (origin) podle lokálního repozitáře (master - základní, hlavní větev repozitáře).

git push -f origin master

Podoba příkazu, který se používá pro fixaci chyb při aktualizaci.

git pull origin master

Aktualizuje stav lokálního repozitáře podle vzdáleného repozitáře.

git log

Vypíše přehled jednotlivých commitů.

git log --oneline

Vypíše zkrácený, jednořádkový přehled commitů.

git clone https://github.com/uživatelské jméno/název vzdáleného repozitáře.git

Vytvoření klonu vzdáleného repozitáře na lokálním počítači.
